Selectors

Selectors
Order Number Selector Description Detail
If there is more than one set of rules that apply to an element then the rule of specificity and cascading will determine which styling to use
#1 Important Applied after any value to indicate that it should be more important than other rules that apply to this element Add "!Important" after an element to indicate that it should overrule any other styling. Use sparingly.
#2 ID Selector Matches an element assigned an "ID" name in HTML
  • Represented by the HASH symbol "#"
  • Highest ranking in stling
#3 Class Selector Matches an element assigned an "Class" name in HTML
  • Represented by the period symbol "." also referred to as the full stop symbol
  • Example: ".note". This would apply to any line of code that is identified as "class='note'"
  • More specific to apply style to a class inside of a type use the type full stop symbol and class name. Example: "p.note". This would apply to class of note inside a paragraph type.
#4 Type Selector Applies style to types of HTML elements such as h1 and p
  • Example: h1 type would apply styling to all h1 types that do not have a class or id where there is already a more specific styling applied
  • Can apply styling to multiple types at one time by adding each type followed by a space Example: h1 p this would apply to all h1 and p types
#5 Child Selector An element that is the child of another element Example: "li>a" targets any a element that is the child of an li element but not other a elements in the page
#6 Descendant Selector Targets all elements that fall under the selector of the type. Example: p a will target an a element that sits inside a p element. Even if there are other elements nested between them.
#7 Adjacent Sibling Selector Targets an element that is the next sibling of another Example: h1+p will target the first p element after any h1 element but not other p elements
#8 General Sibling Selector Targets an element that is a sibling of another, although it does not have to be the direct preceding element. Example: h1~p would target any p element that followed an h1 element.
#9 Last Selector If the two selectors are identical the last selecter will take precedence. Allow Cascading to assist with specificity over using the important selector. Use previous specificity guideline to style a page.
#10 Universal selector Applies to all elements on a page or document. Represented by the asterisk or "*" symbol Example: * {} targets all elements on the page.

Cascading

Cascading Rules
Rule Detail Example
Last If two selectors are identical, the latter of the two will take precedence. If there are two p selectors, the last p style will apply to the element
Specificity If one selector is more specific than others, the more specific rule will take precedence over more general ones. h1 is more specific than *, p b is more specific than p, p#intro is more specific than p.
Important You can add !important after any property value to indicate that it should be considered more important than other rules that apply to the same element. background-color: blue !important;
Use !important as little as possible. Try using specificity and cascading to style a page before turning to !imporant.
